# 원작료 정산서 자동화 프로그램

테라핀 / 수성웹툰의 매월 원작료 정산서를 **생성 → PDF·비밀번호 → 메일 초안**까지
한 번에 처리하는 독립 실행형 프로그램입니다.

## 불변 3원칙
1. **원본 무수정** — NAS 원본/폴더를 직접 읽거나 수정하지 않음. 직전월 정산서는 `inbox/`에 첨부된 *사본*만 읽음.
2. **무발송** — 메일은 `.eml` 초안까지만. 자동 발송 없음.
3. **검수 게이트** — 단계마다 manifest 로그를 남기고 멈춤. 사람이 검수 후 다음 단계.

## 폴더 구조
```
원작료정산_자동화/
├─ config.py          설정·규칙·불변원칙 (회사/경로/증빙구분/비밀번호 그룹)
├─ daterules.py       날짜 규칙 (월간/분기/반기)
├─ settlement.py      정산서 생성 (원본 복제 + 수식→값 + 날짜 + 증빙구분)
├─ securepdf.py       PDF 변환 + 비밀번호
├─ maildraft.py       메일 초안(.eml) 생성
├─ pipeline.py        메인 오케스트레이터 (단계별 실행)
├─ 원작료정산_마스터.xlsx   업체/이메일/MG/이월/증빙구분 마스터
├─ 정산서_양식_기초.xlsx    신규 업체용 표준 템플릿
├─ inbox/             ← 담당자가 직전 발행 정산서 사본을 여기에 둠
└─ output/
   ├─ 정산서/   생성된 정산서(xlsx)
   ├─ PDF/      암호화 PDF
   ├─ 메일Draft/ 회사별 .eml
   └─ logs/     단계별 manifest(csv)
```

## 사용법
```bash
# 1) 정산서 생성
python pipeline.py --company 테라핀 --year 2026 --month 5 --stage settle
# 2) PDF + 비밀번호
python pipeline.py --company 테라핀 --year 2026 --month 5 --stage pdf
# 3) 메일 초안
python pipeline.py --company 테라핀 --year 2026 --month 5 --stage mail

# 한 번에(검수 게이트에서 멈춤):
python pipeline.py --company 테라핀 --year 2026 --month 5 --stage all

# 수성웹툰:
python pipeline.py --company 수성웹툰 --year 2026 --month 5 --stage all
```

## 매월 실행 절차
1. 프로그램이 산출한 **필요 파일 목록**을 보고, 담당자가 직전 발행 정산서 사본을 `inbox/`에 둠.
2. `--stage settle` 실행 → `output/정산서/` 확인(검수).
3. `--stage pdf` 실행 → 비밀번호 적용 확인(검수).
4. `--stage mail` 실행 → `.eml`을 Outlook/MailPlug에서 열어 검수 후 발송.

## 확정 규칙 (코드 반영)
- **지급대상** = 마스터 작품Alias 등록 + 상태≠해지/종료
- **정산금액** = ROUND_HALF_UP(정산기준순매출 × RS율)
- **날짜**: 네이버=대상월말일/익월8/익월15, 리디=분기말익월25/익월말일/익익월10,
  기타사업자=익월5/10/15, 개인=지급 익월15, **카카오="-"**
- **모든 날짜 YYYY-MM-DD** (미국식 금지, 한글 '년/월' 월표시는 보존)
- **증빙구분**(마스터): 세금계산서(과세) / 계산서(면세, 부가세0) / 혼합(에이템포: 국내계산서+해외세금계산서)
- **MG 억제**: 잔여=MG총액−누적−당월, 잔여>0이면 공급가0/「MG 차감중」
- **이월**: 직전 정산서 정산기준순매출 SUM × RS (이월 마스터에 대상만 표시)
- **비밀번호**: 사업자=사업자등록번호10자리 / 개인=생년월일6자리
  · 그룹 공통: 카카오 4곳=카카오 대표번호, KW북스·예원북스=KW북스 대표번호
- **부가시트 제거**: KW유통·만타·MG종료·해지·합병·요약 등은 발송 파일에서 삭제

## 별도 사이클 (월간 배치 제외 — 미리 생성, 발송 시 최종 확정)
